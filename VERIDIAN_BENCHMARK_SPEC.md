# Veridian Benchmark Specification

## Purpose

This specification defines how Veridian capabilities are objectively measured.

The goal is to replace architectural claims with reproducible evidence.

Benchmarks must be repeatable, measurable, and independently verifiable.

---

# Benchmark Principles

Every benchmark must define:

- Claim
- Test scenario
- Inputs
- Expected outcome
- Pass criteria
- Metrics
- Evidence
- Reproducibility

# Measurement Rules

Every benchmark should report:

- Detection Rate
- False Positive Rate
- False Negative Rate
- Success Rate
- Failure Rate
- Median Latency
- 95th Percentile Latency
- Replay Accuracy
- Evidence Completeness

---

# Success Criteria

Every published benchmark must include:

- Methodology
- Test data
- Configuration
- Results
- Reproduction steps
- Limitations

Benchmarks without methodology should not be considered valid evidence.
